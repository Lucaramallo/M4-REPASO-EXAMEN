<h2>Elegir la opción correspondiente</h2>
<p>1) Al momento de usar clases fuertemente tipadas en Scala: </p>
<p>Opciones: </p>
<p>a- Si el dato ingestado no es acorde con los tipos definidos, el lenguaje lo infiere y lo carga</p>
<p>b- Si el dato ingestado no es acorde con los tipos definidos, arroja un error en tiempo de compilación</p>
<p>c- Es igualmente performante que el uso de un Dataframe, donde la estructura de datos se infiere</p>
<p>2) En Big Data, la característica <code>variedad</code> consiste en: </p>
<p>Opciones: </p>
<p>a- El establecimiento de un flujo de datos masivo y continuo</p>
<p>b- Poder gestionar y canalizar grandes volúmenes de datos</p>
<p>c- Poder trabajar con datos estructurados y no estructurados</p>
<p>3) La <code>tolerancia a particiones</code> es un concepto relacionado con: </p>
<p>Opciones: </p>
<p>a- Las particiones lógicas en los discos duros del cluster</p>
<p>b- Que se puedan utilizar particiones en lugar de índices para las tablas</p>
<p>c- Que el sistema disponibiliza información aún habiendo tenido fallas de red</p>
<p>d- Que el sistema pueda gestionar particiones lógicas en los discos duros</p>
<p>e- Que la administracion del sistema esté particionada</p>
<p>4) El Teorema CAP dice que: </p>
<p>Opciones: </p>
<p>a- Un sistema SQL tiene la característica de ser consistente</p>
<p>b- Un sistema No-SQL puede ser tolerante a particiones y consistente</p>
<p>c- No es posible para ningún sistema de bases de datos garantizar al mismo tiempo consistencia, disponibilidad y tolerancia a particiones </p>
<p>d- Un sistema SQL puede ser consistente y estar disponible</p>
<p>e- Un sistema SQL o No-SQL no puede garantizar al mismo tiempo consistencia, disponibilidad y tolerancia a particiones</p>
<p>5) ¿Cuáles son las características fundamentales de los motores de bases de datos No-SQL? </p>
<p>Opciones: </p>
<p>a- Consistencia eventual, aislamiento y durabilidad</p>
<p>b- Uso de particiones, escalabilidad horizontal y casos de uso específicos</p>
<p>c- Tolerancia a particiones, escalabilidad horizontal y casos de uso específicos</p>
<p>d- Tolerancia a particiones, consistencia horizontal y casos de uso de negocio</p>
<p>6) Respecto al Data Lake, ¿cuál de las siguientes afirmaciones es correcta?: </p>
<p>Opciones: </p>
<p>a- Es una mejora del Data Warehouse, ya que consiste en tener más capacidad de hardware para la gestión de grandes caudales de datos</p>
<p>b- Es un repositorio unificado de datos estructurados y no estructurados que prioriza el almacenamiento de los datos en su formato original, para posteriormente ser procesados de acuerdo a la demanda</p>
<p>c- Es un sistema de ETL distribuido</p>
<p>7) En un Data Warehouse los datos se organizan bajo un modelo relacional. Sin embargo: </p>
<p>Opciones: </p>
<p>a- En los últimos años, con la evolución de la tecnología de redes y la capacidad de cómputo, ha surgido un nuevo tipo de dato, no estructurado, que requiere de un enfoque distinto al tradicional a la hora de ser gestionado</p>
<p>b- El Data Warehouse, como tal, no está contemplado entre las herramientas de Big Data</p>
<p>c- Este esquema ha dejado de usarse en los últimos años</p>
<p>8) ¿Cuál de las siguientes afirmaciones NO es correcta? </p>
<p>Opciones: </p>
<p>a- Cada DataNode tiene un sistema de archivo local al cual no puede acceder el nodo maestro.</p>
<p>b- El NameNode tiene la tarea de balancear la carga sobre el cluster a la hora de almacenar un nuevo archivo.</p>
<p>c- El NameNode optimiza el ancho de banda y balancea la carga de procesamiento.</p>
<p>9) En un cluster que tiene un factor de réplica de 3 y un tamaño de bloque de 64 MB, pero sólo 3 nodos, se guarda un archivo de 500 MB ¿Qué espacio ocupa ahora el archivo original? </p>
<p>Opciones: </p>
<p>a- 500 MB</p>
<p>b- 1500 MB</p>
<p>c- 64 MB</p>
<p>10) Si se considera que una tabla dentro de Hive va a ser consultada mediante consultas de agregación (Avg(), Sum(), Max(), etc): </p>
<p>Opciones: </p>
<p>a- Conviene que esté almacenada en formato Avro</p>
<p>b- No es Hive la herramienta adecuada, sería mejor elegir un motor No-SQL</p>
<p>c- Conviene que esté almacenada en formato Parquet</p>
<p>11) En caso de realizar una operación de tipo DROP TABLE en Hive: </p>
<p>Opciones: </p>
<p>a- No está habilitada tal funcionalidad</p>
<p>b- Si la tabla es "External" se eliminan solo los datos fuente</p>
<p>c- Si la tabla es "Managed" se elimina tanto la metadata como los datos fuente</p>
<p>d- Si la tabla es "Managed" se elimina la metadata pero no los datos fuente</p>
<p>12) Dentro de la arquitectura <code>docker</code> encontramos: </p>
<p>Opciones: </p>
<p>a- Contenedores, imágenes, volúmenes y redes</p>
<p>b- Imagenes, archivos YML, herramientas de Big Data y redes</p>
<p>c- Máquinas virtuales de poca capacidad de almacenamiento</p>
<p>d- Contenedores, servicios, volúmenes y redes</p>
<p>e- Contenedores y máquinas virtuales</p>
<p>13) Con respecto a <code>docker</code>, podemos afirmar que: </p>
<p>Opciones: </p>
<p>a- Los procesos se ejecutan de forma nativa</p>
<p>b- Los servicios de docker pueden llegar a consumir más recursos de los que se les permite</p>
<p>c- Implementa máquinas virtuales</p>
<p>14) ¿Cuál es la estructura de datos fundamental de <code>Apache Spark</code>? </p>
<p>Opciones: </p>
<p>a- Dataset, que solo está disponible para Scala y Java</p>
<p>b- GraphX, que permite la manipulación de datos en formato de grafos</p>
<p>c- RDD (Resilient Distributed Dataset)</p>
<p>d- Datasets y Dataframes</p>
<p>5- Tablas y Grafos</p>
<p>15) Kafka es un sistema de mensajes distribuido: </p>
<p>Opciones: </p>
<p>a- Que utiliza el patrón productor-consumidor y maneja el concepto de tópico para categorizar esos mensajes</p>
<p>b- Que solo se puede utilizar para procesamiento streaming</p>
<p>c- Que monitorea un proceso streaming</p>
